1)피씨에 깃 설치가 되어 있어야 함
확인법 : cmd치고 도스에서 git이라고 입력

2)형상관리 틀(버전관리)
git : 버전관리 툴로써 개발자는 자신만의 코드 작성시 브랜치(branch) 단위를 파생하여 작업을 진행
      추후 버전으로 사용할지 버릴지를 결정할 수 있다
github : git의 cloud화 시켜놓은 웹서버인데, 단 전세계 모든 개발자들간 소스가 공유될 수 있으므로
         일종의 커뮤니티로 발전

3)git을 이용하여 작업하는 방법

a) 대상 프로젝트 폴더에 git 관리 디렉토리를 만든다
   git init
   git 디렉토리 안에는 git 현재또는 전체 git에 대한 설정정보가 들어있다
4)github에서 클라우드 프로젝트를 생성하여 현재 작업중인 로컬 프로젝트와 연결

a) 현재 작업중인 프로젝트의 구성 파일드을 확정짓자
   즉 버전을 만들자(개발하려는 코드가 완성된 시점)
b) github주소를 환경설정 파일에 등록
    현재 개발중인 프로젝트와 1:1 대응하는 클라우드의 프로젝트를 의미

git init
git add .
git commit -m ""
git push -u 주소 master

만일 주소가 너무 길면 줄여서 알리아스 붙인다
>git remote add origin 나의 깃헙주소 
오리는 예 ) git push -u origin master
